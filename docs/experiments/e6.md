# E6 — Profundidad de red y learning-rate scheduling

## Motivación

---

`metodologia.md` define E6 como un experimento de comparación controlada sobre el surrogate Heston de referencia. La pregunta experimental es si modificar la profundidad de la red o introducir un learning-rate scheduler mejora la calidad de aproximación del surrogate, especialmente en `MAE_Delta`, sin degradar de forma relevante `MAE(C/K)`.

A diferencia de E2, que compara funciones de activación manteniendo fija la arquitectura, y de E5, que compara pérdidas manteniendo fija la arquitectura y el dataset, E6 estudia dos fuentes distintas de mejora potencial: capacidad del modelo (más o menos profundidad) y régimen de optimización (scheduler de learning rate). La métrica primaria es `MAE_Delta` por bin; `MAE(C/K)` se mantiene como control para distinguir mejoras reales en derivadas de posibles sacrificios en el nivel del precio.

## Configuración del experimento

---

Sujetos:

- `results/checkpoints/H-3/` como baseline.
- `results/checkpoints/H-7-shallow/` como variante con menor profundidad.
- `results/checkpoints/H-8-deep/` como variante con mayor profundidad.
- `results/checkpoints/H-9-lr-schedule/` como variante con misma arquitectura que el baseline y scheduler de learning rate.

Arquitecturas propuestas:

- H-3: MLP `4 x 128`, activación Swish.
- H-7-shallow: MLP `2 x 128`, activación Swish.
- H-8-deep: MLP `6 x 128`, activación Swish.
- H-9-lr-schedule: MLP `4 x 128`, activación Swish, con `ReduceLROnPlateau`.

Se mantiene fijo, salvo indicación contraria:

- misma familia de modelo (Heston),
- mismo dataset de entrenamiento,
- mismo dataset de validación,
- mismo test balanced,
- misma activación,
- misma función de pérdida,
- mismo batch size,
- misma semilla de entrenamiento, si es viable,
- mismo criterio de selección del mejor checkpoint por validación.

El scheduler propuesto para `H-9-lr-schedule` es `ReduceLROnPlateau`, con reducción multiplicativa del learning rate cuando la validación deja de mejorar durante un número fijado de épocas.

Test set:

- `data/heston_test_125k_balanced_delta.npz`

Métricas reportadas:

- `MAE_Delta` medio por bin (métrica primaria),
- `MAE(C/K)` medio por bin (control),
- `p95` de `MAE_Delta`,
- número de parámetros,
- tiempo de entrenamiento total,
- época del mejor checkpoint.
